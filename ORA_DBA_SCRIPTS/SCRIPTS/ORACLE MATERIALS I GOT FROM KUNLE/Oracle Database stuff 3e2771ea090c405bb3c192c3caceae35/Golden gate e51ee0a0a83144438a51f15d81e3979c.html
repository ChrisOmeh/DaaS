Golden gate

Streams pool size in the db SGA affects the streams process and golden gate process

Ensure the ld_library path points to oracle home in the environment or bash profile

Should be able to edit in the DBOPTIONS parameter in ggsci.

Should be worth considering when gg process is having resource issues (memory etc)

The below steps can be done after installing the binaries (on the source)

To setup golden gate, the database must be in archive mode.

for archive mode, set the db recovery file dest first and its size

Shutdown immediate;

Startup mount ;

Alter database archivelog;

Alter database open;

Alter pluggable database all open;

You must also enable force logging and supplemental logging

For the supplemental logging the common logging to activate is the all columns supplementa lying

Alter database force logging

Alter database add supplemental log data (all) columns;

(run at off peak because it can take a full at peak period )

Enable golden gate replication

show parameter enable_golden_replication

Alter system set enable_golden_replication=true scope=both;

Enable flashback

Alter database flashback on;

Create GG user and tablespace for golden gate user

Apparently crest tablespace on both plug gable and container

Grant connect resource select any dictionary and table and many others to the user.

Run marker_setup.sql and follow the prompt

Run role_setup.sql and follow the prompt

Run @sequence

run both on container level

Both sql scripts are found in the golden gate home
