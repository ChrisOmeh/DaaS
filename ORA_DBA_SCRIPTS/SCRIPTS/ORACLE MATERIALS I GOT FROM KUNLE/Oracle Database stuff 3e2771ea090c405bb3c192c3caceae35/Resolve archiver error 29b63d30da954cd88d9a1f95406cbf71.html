Resolve archiver error

Archiver error happens when the db_recovery_file_dest is full and does not have any free space.

A very common Oracle database error that is basically telling you that you have run out of logical or physical space on the mount, disk, or your db_recovery_file_dest location that holds your archivelogs. When you get this error your database hangs and will not allow anyone besides admin level users to log into the database to perform maintenance to fix the problem.

The first step to fixing this issue would be to log into your server and see if you have run out of physical space on one of your disks or mounts.

If you have run out of physical space you have a few options.

You can either point your archivelogs to write to a new disk or mount with sufficient space,

you can backup your archivelogs and delete input, or

you can just delete the archivelogs if you wonâ€™t need them to restore your database.

I would recommend using rman for either the backup or delete methods as this will keep your catalog of backups up to date and clean.

To solve, clear archive logs

# to delete archive log older than a particular day.
# run in RMAN

run{
crosscheck archivelog all;
delete noprompt archivelog all completed before 'sysdate-4';
}
exit;

# to backup archive log
# run in RMAN

run {
BACKUP ARCHIVELOG FROM TIME 'SYSDATE-30' UNTIL TIME 'SYSDATE-7';
}

Forcefully delete archivelogs 

delete force noprompt archivelog all completed before 'sysdate - 3';
